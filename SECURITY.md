# Security Policy

## Supported Versions

| Version | Supported         |
| ------- | ----------------- |
| 1.x     | ✅ Active support |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues.**

If you discover a security vulnerability in OptiCore React Native, report it responsibly:

**Email**: [dev.m.elshenawy@icloud.com](mailto:dev.m.elshenawy@icloud.com)  
**Subject**: `[SECURITY] opticore-react-native - <brief description>`

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

### What to Expect

- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- **Fix & disclosure** coordinated with you before public release

---

## Scope

Security issues relevant to this library include:

- Token/credential exposure through logs or storage APIs
- Insecure default configurations
- Dependency vulnerabilities with direct impact
- Data leakage through error messages or state

Out of scope: vulnerabilities in consumer app code that uses this library.

---

## Disclosure Policy

We follow [Responsible Disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure). Once a fix is released, we will publish a security advisory on GitHub.
